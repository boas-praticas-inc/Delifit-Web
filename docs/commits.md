# Padrão de commits

Use commits curtos, objetivos e no presente.

## Formato sugerido
`tipo: descrição curta`

## Tipos comuns
- `feat`: nova funcionalidade
- `fix`: correção de bug
- `docs`: documentação
- `refactor`: refatoração sem mudar comportamento
- `test`: testes
- `chore`: tarefas de manutenção
- `style`: ajustes de formatação ou estilo

## Exemplos
- `feat: criar CRUD de restaurantes`
- `fix: corrigir validação do formulário de usuario`
- `docs: atualizar README do frontend`
- `refactor: organizar services de usuarios`

## Boas práticas
- Use verbo no imperativo
- Prefira mensagens curtas e claras
- Evite commits muito genéricos como "update" ou "fixes"
- Se possível, faça um commit por mudança lógica
